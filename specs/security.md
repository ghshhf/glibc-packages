# SkyNet SSI 安全模型规范

> **System Standard Security Model** — 分层安全架构与权限体系。

---

## 1. 安全层级

```
┌──────────────────────────────────────────────┐
│  L5: 应用层安全                                │
│  • WASM 沙箱隔离 (每个组件独立沙箱)              │
│  • 无直接系统调用                               │
│  • 声明式权限模型                               │
├──────────────────────────────────────────────┤
│  L4: 服务层安全                                │
│  • 服务间 ACL                                  │
│  • 接口级权限检查                               │
│  • 速率限制                                     │
├──────────────────────────────────────────────┤
│  L3: 内核层安全                                │
│  • 内存保护 (WASM 线性内存隔离)                  │
│  • IPC 审计                                     │
│  • 组件签名验证                                 │
├──────────────────────────────────────────────┤
│  L2: 硬件层安全                                │
│  • TPM 2.0 / Secure Enclave                   │
│  • 安全启动链                                   │
│  • 硬件密钥存储                                 │
└──────────────────────────────────────────────┘
```

## 2. 权限模型

所有组件通过 manifest.json 声明所需权限，运行时强制检查。

```json
{
  "permissions": [
    {
      "interface": "SSI-NET",
      "operations": ["fetch"],
      "restriction": "outbound-only"
    }
  ]
}
```

## 3. 信任链

```
硬件信任根 (TPM/Enclave)
    ↓ 测量
Bootloader
    ↓ 验证签名
Kernel + WASM Runtime
    ↓ 验证签名和权限
System Components (.swbn)
    ↓ 验证签名和权限
User Applications (.swbn)
```

## 4. 加密原语

| 功能 | 算法 | 密钥长度 |
|------|------|---------|
| 组件签名 | Ed25519 | 32 bytes |
| P2P 加密 | XChaCha20-Poly1305 | 32 bytes |
| 密钥交换 | X25519 + BLAKE2b | 32 bytes |
| 内容哈希 | SHA256 / BLAKE3 | 32 bytes |
| 节点 ID | SHA256(公钥) | 32 bytes |
